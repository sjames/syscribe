The `syscribe template Diagram` output (generated by the `template` subcommand) shall
include `pumlMode` and `pumlFile` as commented-out example lines in the YAML frontmatter
block, alongside the existing `svgMode`/`svgFile` examples.

## Required template excerpt

```yaml
# svgMode: companion      # opt-in SVG companion; default path: <stem>.svg
# svgFile: diagrams/foo.svg
# pumlMode: companion     # opt-in PlantUML companion; default path: <stem>.puml
# pumlFile: diagrams/foo.puml
```

The comment lines shall appear immediately after the `svgMode`/`svgFile` comment block so
that the two companion modes are visually grouped.

## Rationale

Authors discover the SVG companion feature through `syscribe template Diagram`. Placing
`pumlMode`/`pumlFile` in the same section ensures authors discover the PlantUML companion
capability through the same workflow rather than having to consult the format specification
or documentation site. Commented-out lines keep the generated stub minimal while still
advertising the capability.
