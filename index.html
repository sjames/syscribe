    Syscribe is a plain-text modeling format that maps SysMLv2 semantics onto Markdown files with YAML frontmatter. Version-controlled, diff-friendly, LLM-readable — with a built-in validation engine and web browser.

    § 01 — WHY
    Systems models that live in your repository.

    Traditional MBSE tools keep models in proprietary binary formats: opaque to diff, hostile to code review, and invisible to the LLMs that increasingly assist engineering work. Syscribe puts the model in plain text where it belongs — readable in any editor, diffable in any VCS, and parseable by any tool.

    The format enforces SysMLv2 semantics through YAML frontmatter schema validation and cross-reference resolution, so you get modeling rigour without a heavyweight IDE. Requirements carry stable REQ-* IDs that survive renames. TestCases carry TC-* IDs traceable back to requirements. ADRs carry ADR-* IDs that justify every decomposition decision.

    Because every element is a separate file, it has a stable URL in any git host. A GitHub permalink to Requirements/Safety/REQ-ENG-SAFE-001.md points to that exact requirement at that exact commit — forever. JIRA tickets, Confluence pages, code review comments, and CI reports can all link directly to a specific requirement, test case, or architecture decision, pinned to a branch tip, release tag, or commit hash.

    § 02 — CAPABILITIES
    Everything a systems model needs.
    STRUCT
    Structural hierarchy
    PartDef, ItemDef, PortDef, InterfaceDef, ConnectionDef — full SysMLv2 structural decomposition in plain files.
    REQ
    Native requirements
    Stable REQ-* IDs, lifecycle status, SIL/ASIL fields, normative text, derivation trees, domain classification.
    TC
    Test cases
    TC-* IDs, L1–L5 test levels, Gherkin scenarios, testFunctions cross-reference, active/retired lifecycle.
    ADR
    Architecture decisions
    ADR-* IDs with proposed/accepted/deprecated/superseded status. Every requirement decomposition cites an accepted ADR.
    DIAG
    Multiple diagram kinds
    BDD, IBD, StateMachine, Requirement diagrams via structured frontmatter. Mermaid diagrams via fenced code blocks.
    OP
    Interface operations
    Synchronous operations and async receptions on PortDef and InterfaceDef — typed parameters, return types, isQuery flag.
    ALLOC
    Allocations
    Function-to-component and requirement-to-component allocation with cross-reference validation and domain checks.
    PLE
    Product-line engineering
    FeatureDef/Configuration elements with appliesWhen conditioning — variant modeling without a separate tool.
    LIVE
    Live browser
    Axum server with file-system watcher. HTMX partial updates — tree navigation, element detail, and diagrams without a page reload.
    § 03 — QUICK START
    Up and running in four steps.
    STEP 01 — CLONE & BUILD
    Build the toolchain

    Requires Rust stable. One command builds the validator CLI and the web server.

    # clone the repo git clone github.com/sjames/syscribe cd syscribe # build both binaries cargo build --workspace
    STEP 02 — VALIDATE
    Run the validator

    Point it at any of the three demo models. The full report covers element inventory, requirements traceability, and all findings.

    # Engine ECU (ISO 26262 / 21434) syscribe -m model_auto/ # SIL 4 railway interlocking syscribe -m model_sil/ # findings only syscribe -m model_auto/ validate
    CLI reference
    STEP 03 — BROWSE
    Open the web browser

    The server watches for file-system changes and reloads automatically. The canvas gives an interactive graph view of the whole model.

    # start the server syscribe-server -m model_auto/ # open in browser http://localhost:3000/ http://localhost:3000/canvas
    Browser docs
    STEP 04 — QUERY
    Explore with queries

    Trace a requirement end-to-end, find what a component satisfies, or get the next available ID before adding a new element.

    # full trace for a requirement syscribe -m model_auto/ \ trace REQ-ENG-SAFE-001 # what does this component satisfy? syscribe -m model_auto/ \ why System::Software::SafetyMonitor # next available ID syscribe -m model_auto/ \ next-id REQ-ENG-SAFE
